## Notes

Deep dive into lifetimes today. This was confusing at first but starting to make sense.

Lifetimes are about ensuring references are valid for as long as they're used. The compiler usually infers them, but sometimes you need to annotate.

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

The `'a` tells the compiler: "the returned reference will be valid for at least as long as both input references."

Key insight: Lifetimes don't change how long values live - they just help the compiler verify that references are valid.
